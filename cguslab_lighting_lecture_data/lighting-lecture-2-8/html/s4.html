Maya 독립 실행 환경 (Standalone)

maya.standalone 모듈을 사용하면 Maya UI 없이 백그라운드에서 스크립트로 Maya 씬 파일을 다룰 수 있습니다. 수십, 수백 개의 파일을 대상으로 동일한 작업을 반복하는 배치(Batch) 처리에 매우 유용합니다.

[개념 정리] PyMEL, maya.cmds, OpenMaya API 2.0

  • PyMEL: pymel.core는 maya.cmds를 객체지향적으로 감싼 래퍼. 노드를 파이썬 객체처럼 다루어 가독성과 생산성이 높음. 대체로 초보자에게 추천.
  • maya.cmds: 가장 널리 쓰이는 절차적 명령형 API. 간단하고 의존성이 적어 스크립트 호환성이 좋음.
  • OpenMaya API 2.0: maya.api.OpenMaya 저수준 API. 가장 빠르고 메모리 효율적이며 대규모 배치 처리/플러그인 개발에 유리. 다만 학습 곡선이 있음.

권장 가이드: 학습/프로토타이핑은 PyMEL → 배포/성능 요구가 커지면 cmds 또는 OpenMaya API 2.0으로 전환

선택된 포인트 라이트 Intensity 2배 (Standalone 배치 버전)

import maya.standalone
import os
import pymel.core as pm

def double_point_light_intensity_with_pymel(target_folder):
    maya.standalone.initialize()
    try:
        for file_name in os.listdir(target_folder):
            if file_name.endswith((".ma", ".mb")):
                file_path = os.path.join(target_folder, file_name)
                pm.openFile(file_path, f=True)

                point_lights = pm.ls(type='pointLight')
                for light in point_lights:
                    light.intensity.set(light.intensity.get() * 2)

                pm.saveFile()
                print(f"{file_name} 처리 완료.")
    finally:
        maya.standalone.uninitialize()

# 예) double_point_light_intensity_with_pymel('C:/maya_batch_test')

PyMEL은 노드를 객체처럼 다루므로 코드가 간결하고 읽기 쉽습니다.

씬의 모든 포인트 라이트 Intensity 2배 (고성능 순회)

import maya.standalone
import maya.api.OpenMaya as om
import maya.cmds as cmds
import os

def double_point_light_intensity_api2(target_folder):
    maya.standalone.initialize()
    try:
        for file_name in os.listdir(target_folder):
            if file_name.endswith((".ma", ".mb")):
                file_path = os.path.join(target_folder, file_name)
                cmds.file(file_path, open=True, force=True)

                it = om.MItDependencyNodes(om.MFn.kPointLight)
                while not it.isDone():
                    obj = it.thisNode()
                    fn = om.MFnDependencyNode(obj)
                    plug = fn.findPlug('intensity', False)
                    plug.setFloat(plug.asFloat() * 2.0)
                    it.next()

                cmds.file(save=True, force=True)
                print(f"{file_name} 처리 완료.")
    finally:
        maya.standalone.uninitialize()

# 예) double_point_light_intensity_api2('C:/maya_batch_test')

OpenMaya API 2.0은 대규모 배치에서 빠르며, 플러그(Plug) 단위 접근으로 세밀한 제어가 가능합니다.

[실전 문제 12-2]

특정 폴더 안의 모든 Maya 씬 파일(.ma, .mb)을 찾아서, 각 씬을 열고 씬 안에 있는 모든 포인트 라이트의 Intensity(강도)를 2배로 올린 뒤, 원본 파일에 그대로 덮어쓰는 배치 스크립트를 작성해보세요.

주의: 원본 파일을 덮어쓰므로, 테스트용 씬 파일을 따로 복사해서 진행하세요.

해답:

import maya.standalone
import maya.cmds as cmds
import os

def batch_process_light_intensity(target_folder):
    maya.standalone.initialize()

    for file_name in os.listdir(target_folder):
        if file_name.endswith(('.ma', '.mb')):
            file_path = os.path.join(target_folder, file_name)
            cmds.file(file_path, open=True, force=True)
            
            point_lights = cmds.ls(type='pointLight')
            for light in point_lights:
                current_intensity = cmds.getAttr(f"{light}.intensity")
                cmds.setAttr(f"{light}.intensity", current_intensity * 2)
            
            cmds.file(save=True, force=True)
            print(f"{file_name} 처리 완료.")

    maya.standalone.uninitialize()
